# Contalyst — Requirements（要件）

> 機能要件 (FR) と非機能要件 (NFR)。各要件は一意 ID を持ち、テスト可能な記述とする。
> 対応ストーリーは [03-stories.md](./03-stories.md)、UI 仕様は [04-ux-design.md](./04-ux-design.md) を参照。

## Intent Analysis Summary（意図分析サマリ）

- **Request type**: 新規プロダクト開発（greenfield）。
- **Core intent**: Docker コンテナ操作の CLI 反復を、カラフルで discoverable な TUI に置き換える。
- **判断基準**: 既存類似 TUI と比較して **UX が最良** であること。
- **Must use**: Go, Bubble Tea。その他は Apache-2.0（Charm 系 MIT は例外として許容 / DR-1）。

優先度表記: **P0** = MVP 必須, **P1** = MVP 望ましい, **P2** = Post-MVP。

## Functional Requirements（機能要件）

### コンテナ (Containers)
| ID | 優先 | 要件 |
|---|---|---|
| FR-C1 | P0 | 全コンテナ（停止中含む）の一覧を、名前/イメージ/状態/ステータス/ポートと共に表示する |
| FR-C2 | P0 | 一覧は一定間隔または Docker イベントに応じて自動更新される |
| FR-C3 | P0 | 選択コンテナのログをリアルタイム追尾 (follow) し、自動スクロールする |
| FR-C4 | P0 | ログは時間窓で勝手に truncate せず、過去ログ＋新規ログを表示する |
| FR-C5 | P1 | ログ内を検索でき、タイムスタンプ表示を切り替えられる |
| FR-C6 | P0 | 選択コンテナを start / stop / restart できる |
| FR-C7 | P1 | 選択コンテナを pause / unpause / kill できる |
| FR-C8 | P0 | 選択コンテナを削除できる（確認ダイアログ経由） |
| FR-C9 | P0 | 選択コンテナ内のシェルへ exec できる（既定シェル→`/bin/sh` フォールバック） |
| FR-C10 | P1 | 選択コンテナの inspect 情報を整形表示する |
| FR-C11 | P1 | 選択コンテナの CPU/メモリ/ネット/IO 統計をリアルタイム表示する（CPU% は delta から算出） |

### イメージ / ボリューム / ネットワーク
| ID | 優先 | 要件 |
|---|---|---|
| FR-I1 | P1 | イメージ一覧（リポジトリ/タグ/サイズ/作成日）を表示する |
| FR-I2 | P1 | イメージを削除 / prune できる（確認ダイアログ経由） |
| FR-V1 | P1 | ボリューム一覧を表示し、削除 / prune できる |
| FR-N1 | P1 | ネットワーク一覧を表示し、削除できる |

### ナビゲーション & 操作 UI
| ID | 優先 | 要件 |
|---|---|---|
| FR-NAV1 | P0 | vim ライク（j/k, h/l, g/G）＋矢印キーで一覧を移動できる |
| FR-NAV2 | P1 | `:` コマンドパレットで Docker の語彙によりリソース種別を切替できる（autosuggest 付き） |
| FR-NAV3 | P0 | `/` でファジーフィルタを行い、`Esc` で解除できる |
| FR-NAV4 | P0 | `Enter` でドリルダウン、`Esc` で戻る。現在位置をパンくずで示す |
| FR-NAV5 | P0 | 画面下部に、現在ビューで有効な単キー操作を示すヒントバーを常時表示する |
| FR-NAV6 | P1 | ヒントバーをコンパクト 1 行モードに切替できる |
| FR-NAV7 | P0 | `?` で現在コンテキストのキーバインド一覧（ヘルプ）を表示する |
| FR-NAV8 | P0 | 破壊的操作（削除/kill/prune）は確認ダイアログを経由し、既定フォーカスは非破壊側 |
| FR-NAV9 | P1 | テーマを切り替えられる |
| FR-NAV10 | P0 | `q` / `Ctrl+C` で安全に終了し、ストリーム購読を解除する |

### 接続・環境
| ID | 優先 | 要件 |
|---|---|---|
| FR-E1 | P0 | `DOCKER_HOST` 等の環境変数 / デフォルトソケットで Docker に接続する |
| FR-E2 | P0 | API バージョンをネゴシエーションし、複数 Engine バージョンで動作する |
| FR-E3 | P0 | Docker に接続できない場合、原因の分かるエラーを表示し異常終了しない |

## Non-Functional Requirements（非機能要件）

### パフォーマンス / 応答性
| ID | 優先 | 要件 |
|---|---|---|
| NFR-P1 | P0 | 一覧の再描画・更新は体感的に即時（目標 < 60ms）で、入力をブロックしない |
| NFR-P2 | P0 | ログ追尾・統計はメインスレッドをブロックせず goroutine で処理する |
| NFR-P3 | P1 | 数百コンテナ規模でもスクロール/フィルタが破綻しない |

### 信頼性 / 安全性
| ID | 優先 | 要件 |
|---|---|---|
| NFR-R1 | P0 | ストリーム goroutine は context キャンセルでリークなく停止する |
| NFR-R2 | P0 | コンテナ内 / SSH 越し実行でもログ・統計が正しく表示される |
| NFR-R3 | P0 | 破壊的操作は明示確認なしに実行されない |
| NFR-R4 | P1 | Docker 側エラー（権限・存在しない対象等）はクラッシュせず通知する |

### ユーザビリティ / 見た目
| ID | 優先 | 要件 |
|---|---|---|
| NFR-U1 | P0 | ドキュメント未読でも、ヒントバーのみで主要操作に到達できる（discoverability） |
| NFR-U2 | P0 | コンテナ状態が色で即判別できる（running/exited/paused 等） |
| NFR-U3 | P0 | truecolor / 256 色端末でカラフルに、未対応端末でも破綻しない |
| NFR-U4 | P1 | 端末予約キー（Ctrl-D/R/S）にバインドしない |
| NFR-U5 | P1 | 角丸フレーム等の装飾は opt-in にできる |

### 保守性 / 移植性 / ライセンス
| ID | 優先 | 要件 |
|---|---|---|
| NFR-M1 | P0 | Docker SDK アクセスを `dockerx` に隔離し、将来の moby 移行を局所化する |
| NFR-M2 | P0 | 単一バイナリで配布でき、外部ランタイムを要しない |
| NFR-M3 | P0 | 依存は Charm 系=MIT、その他=Apache-2.0 のみ。コピーレフト不採用（DR-1）|
| NFR-M4 | P1 | Linux/macOS のターミナルで動作（Windows は WSL 経由） |
